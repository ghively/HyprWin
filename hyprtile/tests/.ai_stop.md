# AI Agent Guidance — tests/

## Purpose
Integration tests for all modules.

## Before Adding Tests
1. Follow the existing test group organization (rect, layout, window, workspace, etc.).
2. Use real data (actual rect values, actual config strings) — no mocks needed for pure functions.
3. Name tests descriptively: `test_[module]_[behavior]_[condition]`.
4. Fixtures are in `tests/fixtures/` — add new `.toml` files for config tests.

## Running Tests
```bash
cargo test
cargo test --test integration_tests
cargo test test_dwindle  # run specific test
```

## Test Coverage Goals
- Layout algorithms: test empty, single, two, many windows.
- Window state: test all transitions and toggles.
- Config: test parsing, validation, roundtrip serialization.
- IPC: test all command variants serialize/deserialize correctly.
