# AI Agent Guidance ‚Äî src/window/

## Purpose
Window state machine, filtering, and rule engine.

## Key Files
- `model.rs` ‚Äî `Window` struct and `WindowState` enum (5 states).
- `filter.rs` ‚Äî Window eligibility filtering (system, tool, UWP, cloaked).
- `rules.rs` ‚Äî Regex-based rule matching engine.
- `mod.rs` ‚Äî `WindowManager` registry.

## Before Modifying
- `WindowState` transitions must be reversible (toggle_float, toggle_fullscreen).
- `previous_state` is used for restore ‚Äî always set it before changing state.
- `floating_rect` captures position when floating ‚Äî don't capture minimized positions.
- `RuleEngine` compiles regex once ‚Äî don't recompile on every match.

## Extension Points
- **New window state**: Add variant to `WindowState`, update transitions.
- **New filter**: Add function in `filter.rs`, call in `passes_all_filters()`.
- **New rule match field**: Add to `WindowRule`, update `window_matches_rule()`.

## Critical Invariants
- `WindowManager` is the single source of truth for window metadata.
- `WindowId` equality is identity ‚Äî all collections use it as the key.
- Never use `expect()` in state accessors ‚Äî use `Option` or default fallbacks.
