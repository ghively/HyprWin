# AI Agent Guidance — src/workspace/

## Purpose
Workspace system: per-monitor virtual desktops.

## Key Files
- `model.rs` — `Workspace`, `MonitorWorkspace`, `FocusDirection`.
- `mod.rs` — `WorkspaceManager` with cross-monitor coordination.

## Before Modifying
- Workspaces are lazy-created — `ensure_workspace()` creates on demand.
- `window_to_workspace` and `window_to_monitor` must stay in sync with `Workspace.windows`.
- `active_workspace` is an ID — the actual workspace object is looked up in the Vec.
- Monitor disconnect must redistribute windows to a fallback monitor.

## Extension Points
- **Named workspaces**: Add `name` customization beyond numeric IDs.
- **Workspace rules**: Auto-assign windows to workspaces by rule.
- **Workspace layouts per-monitor**: Store default layout per monitor.

## Critical Invariants
- `get_active_workspace()` must NEVER panic — use `OnceLock` fallback.
- `ensure_workspace()` must return a valid mutable reference.
- After `switch_workspace()`, old workspace windows must be hidden, new ones shown.
