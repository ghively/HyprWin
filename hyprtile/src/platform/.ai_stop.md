# AI Agent Guidance — src/platform/

## Purpose
Win32 API wrappers. This is the only module that calls `unsafe` Win32 APIs.

## Key Files
- `window.rs` — HWND operations, deferred positioning.
- `monitor.rs` — Display enumeration, DPI.
- `events.rs` — WinEventHook registration and classification.
- `dwm.rs` — Desktop Window Manager effects.
- `input.rs` — Global hotkey registration.
- `startup.rs` — Registry auto-start.
- `tray.rs` — System tray icon.

## Before Modifying
- Every Win32 call needs error handling (`GetLastError` or `Result`).
- Prefer `windows-rs` safe wrappers over raw `unsafe` blocks.
- Test on both Windows 10 and Windows 11.
- Document any undocumented APIs (e.g., `SetWindowCompositionAttribute`).

## Critical Safety
- `DeferredPositioner` must always call `commit()` or the `Drop` impl runs.
- `EVENT_SENDER` is a `Mutex<Option<Sender>>` — lock it briefly, never hold across awaits.
- `HOTKEY_ACTIONS` is a global `Mutex<HashMap>` — same rule: lock, access, release.

## Extension Points
- Add new DWM attributes in `dwm.rs`.
- Add new event types in `events.rs` (update `classify_event()`).
- Add new hotkey key mappings in `input.rs`.
