# AI Agent Guidance — src/app.rs (Single File)

## Purpose
Main application coordinator. This is the central nervous system of HyprTile.

## Key Responsibilities
1. Initialize all subsystems (config, monitors, workspaces, window manager).
2. Spawn and manage worker threads (event hook, hotkey loop, TCP server).
3. Run the main event loop, dispatching events to handlers.
4. Apply layouts when state changes.
5. Graceful shutdown: signal threads, join handles, release resources.

## Before Modifying
- `AppState` holds all mutable state — pass references, not copies.
- `process_event()` is the single dispatch point — add new event variants there.
- `handle_hotkey()` is the action dispatch point — add new action strings there.
- `apply_layout()` is called frequently — keep it fast (< 100ms for 50 windows).
- Thread management: every spawn must have a corresponding join on exit.

## Extension Points
- **New event type**: Add to `WindowEvent`, handle in `process_event()`.
- **New action**: Add string match in `handle_hotkey()`, implement method.
- **New lifecycle hook**: Add to `App::new()` (init) or shutdown sequence.

## Critical Invariants
- `running` flag controls the main loop — set it false in `exit()`.
- `event_rx` disconnect causes loop exit — ensure senders are dropped on shutdown.
- `hotkey_shutdown` AtomicBool signals the hotkey thread to exit.
- `tcp_shutdown` Notify signals the TCP server to exit.
- Never call `expect()` on thread join — log errors and continue.
