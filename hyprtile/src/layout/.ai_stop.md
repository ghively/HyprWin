# AI Agent Guidance — src/layout/

## Purpose
Layout engine: BSP tree + 4 layout algorithms.

## Key Files
- `bsp.rs` — Binary space partitioning tree (core data structure).
- `dwindle.rs`, `master_stack.rs`, `monocle.rs`, `grid.rs` — Layout algorithms.
- `gaps.rs` — Gap calculation system.
- `mod.rs` — Layout coordinator, `LayoutType` enum, `LayoutEngine`.

## Before Modifying
- The BSP tree (`Node` enum) is the foundation — test changes thoroughly.
- All layouts must return `Vec<(WindowId, Rect)>` with valid rects (width >= 1, height >= 1).
- Smart gaps behavior: disable inner gaps when only one window.
- Ratios are always in [0.0, 1.0] and represent the left/top portion.

## Extension Points
- **New layout**: Create `src/layout/new_layout.rs`, add `LayoutType` variant, wire into `calculate_layout()`.
- **New gap mode**: Add field to `GapsConfig`, update `effective_gaps()`.
- **Persistent ratios**: Store BSP tree per-workspace (currently rebuilt each pass).

## Common Pitfalls
- Integer division discards remainder — distribute remainder pixels.
- Grid dimensions must satisfy `rows * cols >= window_count`.
- Monocle layout must clamp `focused_idx` to valid range.
