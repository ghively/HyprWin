# AI Agent Guidance — src/ipc/

## Purpose
Inter-process communication: named pipe + TCP for external control.

## Key Files
- `protocol.rs` — JSON request/response types (serde).
- `commands.rs` — Command handlers dispatching to AppState.
- `mod.rs` — Transport layer (NamedPipeServer, TCP listener).

## Before Modifying
- Protocol is length-delimited JSON: 4-byte big-endian length prefix + JSON body.
- All commands are stateless — handler takes `&mut AppState`, returns `IpcResponse`.
- TCP server uses `tokio::select!` with shutdown Notify for clean exit.
- Named pipe is Windows-only — don't break cross-platform compilation.

## Extension Points
- **New command**: Add to `IpcRequest`, handler in `commands.rs`, docs in `IPC_PROTOCOL.md`.
- **New transport**: Add alongside named pipe/TCP (e.g., Unix domain socket).
- **Binary protocol**: Implement alternative to JSON for lower latency.

## Critical Invariants
- `IpcResponse::success()` and `IpcResponse::error()` are the only response constructors.
- Serialization errors must be caught — never let a malformed response crash the server.
- The server must survive client disconnects without crashing.
